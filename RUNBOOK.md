# EuroFest — Operations Runbook (free tier)

The backend runs on the **Firebase Spark (free) plan**. Cloud Functions/Scheduler
(which need the paid Blaze plan) are **not** used; the identical ingest +
notification pipeline runs as a Node script (`functions/`) driven by a **GitHub
Actions cron**. Firestore, Auth, and FCM are all free.

Project: **euro-fest-12345**.

## What's already wired (in this repo)
- `lib/firebase_options.dart` + native config (`google-services.json`,
  `GoogleService-Info.plist`) — generated by `flutterfire configure`.
- App: anonymous Auth on launch; profile/follows/FCM-token/notif-prefs mirrored
  to `users/{uid}`; events + notification inbox read live from Firestore.
- Firestore **rules + indexes deployed**.
- Backend runner: `npm run seed` (catalog → Firestore) and `npm run ingest`
  (pull sources → upsert events → fan-out notifications).
- `.github/workflows/ingest.yml` — cron every 6h + manual dispatch.

## One-time steps you need to do (credential-gated)

### 1. Service-account key (for the runner / CI)
Firebase console → **Project settings → Service accounts → Generate new private
key**. Save it as `functions/service-account.json` (already gitignored).

### 2. Seed the catalog + first ingest (local)
```bash
cd functions
cp .env.example .env          # then fill in keys (see below)
npm install
npm run seed                  # writes artists + venues to Firestore
npm run ingest                # Ticketmaster pull → upsert events → notify followers
```

To also include **Eventim** (Python, via pyventim) in the same dedupe-merged run:
```bash
pip install -r eventim/requirements.txt
cd functions
npm run export-catalog -- /tmp/catalog.json
python ../eventim/fetch_eventim.py --catalog /tmp/catalog.json \
    --out /tmp/eventim_raw.json --markets GERMANY,FRANCE,NETHERLANDS,BELGIUM
EVENTIM_RAW_PATH=/tmp/eventim_raw.json npm run ingest
```
The GitHub Actions workflow does all of this automatically each run.
`.env` needs at minimum:
- `GOOGLE_APPLICATION_CREDENTIALS=./service-account.json`
- `TICKETMASTER_API_KEY=...`  (developer.ticketmaster.com — free)
- `BANDSINTOWN_APP_ID=...`    (Bandsintown partnership)

### 3. GitHub Actions cron
Push this repo to GitHub, then add repo **Secrets** (Settings → Secrets and
variables → Actions):
- `FIREBASE_SERVICE_ACCOUNT` = the full JSON contents of the key
- `TICKETMASTER_API_KEY`, `BANDSINTOWN_APP_ID`
- (optional) `EUROFEST_TM_AFFILIATE`, `EUROFEST_EVENTIM_AFFILIATE`,
  `EUROFEST_SKIDDLE_AFFILIATE`, `EUROFEST_DICE_AFFILIATE`

The workflow then runs every 6h (and via "Run workflow").

### 4. iOS push (when you build for iOS)
FCM on iOS needs an **APNs key**: Apple Developer → Keys → create an APNs key,
upload it in Firebase console → Cloud Messaging → Apple app config. Also enable
**Push Notifications** + **Background Modes → Remote notifications** capabilities
in Xcode. Android push works out of the box via `google-services.json`.

## Upgrading to Blaze later (optional)
If you ever move to pay-as-you-go, `functions/src/index.ts` already exposes the
same pipeline as a scheduled Cloud Function — `npm run deploy:blaze` and retire
the GitHub Actions cron. No app changes.
